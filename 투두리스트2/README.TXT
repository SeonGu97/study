(21.06.10)
- todolist를 다시 만들어 보기로 했다.

처음 만들었을 때에는 유튜브 영상을 보면서 따라서 만들었다.
장점은 큰 문제 없이 만들 수 있다. 
단점은 모르는 부분이 있어도 따라 만들어야 하기 때문에 이해하면서 만들지 못한다.

두번째 만들었을 때에는 처음 만든 부분을 참고하고, 모르는 부분은 검색을 해서 찾고 이해를 하며 만들었다.
또, 첫번째 보다 조금 더 많은 기능을 추가했다.
장점은 모르던 부분을 알 수 있기 때문에 이해를 하면서 만들 수 있다.
단점은 모르는 부분을 혼자 찾아보고, 이해하면서 만드는 과정에 코드가 깔끔하지 못하고, 비효율적 일 수도 있다.
그리고 시간이 많이 걸린다.

세번째 만들 때에는 조금더 코드를 깔끔하고, 효율있게 만들어 볼려고 한다.

오늘은 todolist의 가장 기본적인 구조를 만들었고, text에 내용을 입력해서 출력 버튼을 누르면 text에 입력한 내용과,
checkbox, trash btn이 같이 나오도록 만들었다. 아직 css 추가를 하지 않아서, 생성된 모습 그대로 보여진다.

세번째 만드는 todolist이지만, 조금더 완벽하게 만들고 싶은 욕구가 강하게 생긴다.
todolist를 만족 할 만큼 완성이 되면, 다음으로 만들 것도 생각을 해놨기 때문에, 이번에 확실히 복습을 해서 다음 공부에
많이 도움이 됬으면 좋겠다.

(21.06.11)
- 리스트를 만들고, 생성된 리스트 안에 입력된 텍스트를 로컬스토리지에도 똑같이 저장이 되도록 만들었다.
그리고 로컬에 저장된 문자열 하나 하나에 함수를 실행 시켜서, 새로고침이나 웹을 껏다가 켜도 생성한 리스트가
유지되도록 만들었다.

그리고 리스트 안에 체크박스도 리스트가 최초 생성 됬을때에 체크박스의 checked를 새로운 로컬스토리지 안에
입력되도록 만들었다. 최초 생성이 됬을때의 체크박스 상태는 false이다. 생성된 체크박스를 클릭하면 true가 되는데,
체크박스의 체크 상태가 바뀔때 마다 특정 함수를 실행하도록 만들었고, 그 함수는 체크박스의 상태가 들어있는
로컬스토리지 value를 true와 false로 토글해준다. 그래서 체크박스를 체크하면 로컬스토리지의 value도 같이 바뀌고, 
만약 value의 값이 true라면 체크가 유지되도록 만들었다. 웹을 새로고침 하거나 껏다가 켜도 체크박스의 상태는 
유지가 된다.

(21.06.13)
- 생성된 리스트가 체크가 되었을때 trash버튼을 누르면 리스트가 사라지도록 만들었다.

리스트의 text value가 들어있는 로컬스토리지는 같이 삭제가 되지만, 체크박스의 체크 상태가 담긴 로컬스토리지는
아직 같이 삭제가 안된다.

(21.06.14)
- 이전에 리스트를 삭제하는 부분에서 오류를 수정했다.

그리고 체크박스의 체크 상태가 담겨 있는 로컬스토리지도 리스트 삭제와 동시에 같이 삭제 되도록 만들었다.
이로써 수동으로 리스트를 클릭해서 삭제하는 기능은 완벽하게 구현이 됬다.

todolist를 3번째 만들고 있지만, 조금 헷갈려서 헤메긴해도 과거와는 만드는 속도 차이가 월등히 빨라졌다.
그래도 계속 부족한 점을 느낀다. 더 열심히 해야겠다.

(21.06.14)
- select all btn을 누르면 리스트가 모두선택/모두해제 되는 기능을 추가했다.

생성된 체크박스 개수와 체크된 체크박스의 개수를 비교해서, 두개의 개수가 같다면 체크박스는 모두 해제되고,
두개의 개수가 다르다면 모두 선택 된다. 물론 모두 선택/해제 됐을때에 로컬스토리지도 같이 변경이 된다.

(21.06.15)
- clear 버튼 기능을 추가했다.

trash 버튼으로 체크된 리스트를 하나씩 제거가 가능하지만, 체크된 여러개의 리스트를 한번에 삭제 할 수 있도록
편의를 위해 만들었다. 중간에 만들면서 몇개의 오류를 수정하면서 만들었기 때문에 아마도 수정은 안해도 될 것 같다.

이제 기능구현은 하단의 all, completed, uncompleted 버튼만을 남기고 있다. 완성이 된다면 css를 추가, 수정해서
todolist의 외형을 좀 더 이쁘게 꾸밀 생각이다.

(21.06.16)
- option 버튼들의 따른 리스트 분류를 완성했다.

all 버튼을 누르면 체크된/안된 리스트 모두가 화면에 보여지고,
completed 버튼을 누르면 체크 된 리스트만 보여지고,
uncompleted 버튼을 누르면 체크 안된 리스트만 보여진다.

그리고 최초 리스트 생성시 체크박스의 상태는 false인데, completed 버튼을 누르고 체크된 리스트만 보여질때,
리스트를 추가해도 completed에 보여지지 않고 all, umcompleted에 보여진다.

또, completed를 누르면 체크된 리스트만 보여지는데, 이때 이미 체크된 리스트를 한번더 체크하면 해제가 되면서
completed에서 제외가 된다. 반대도 마찬가지이다.

그리고 내가 completed, uncompleted를 누르고 웹을 껏다 키거나 새로고침 하면 리스트가 모두 보여지는데(all 상태),
로컬스토리지를 하나 추가해서 내가 누른 버튼의 value가 입력 되도록 하고, 웹을 껏다가 키거나 새로고침 해도
이전에 클릭해 놓은 버튼의 상태로 유지가 된다. (예로 마지막에 completed 버튼을 누르고 웹을 껏다면 다시 켰을때
completed가 유지되어 보여진다.)

다음은 마지막이 될 것 같다. 만들 기능은 생성 된 리스트의 개수를 확인 할 수 있는 부분을 만들고, 스타일을 추가해서
todolist를 꾸밀 생각이다.

(21.06.17)
- todolist를 완성했다.

option 버튼에 따른 리스트 개수를 보여지게 만들고, option버튼과 리스트를 체크했을때 스타일을 추가했다.
리스트를 생성하거나 지워도 리스트의 개수가 갱신되어 보여진다.

todolist를 마무리 하면서 느낀점은 javascript로 만들면 코드가 커져서 라이브러리나 프레임워크를 사용 해보고
싶긴하다. 하지만 개인적인 생각으로는 javascript 기초를 탄탄하게 다져서 라이브러리나 프레임워크를 써도
금방 이해 할 수 있겠지 라는 생각으로 javascript를 중점으로 공부를 하는 중 이다.

그리고 만들면서 최대한 함수를 효율있게 짜서 재사용 할 수 있도록 하는게 코드의 양과 질을 올리는데 중요 한 것 같다.

todolist를 만들면서 알고있던 부분을 조금 더 확실히 하는 느낌이 있었고, 또 겉 핡기로 알고 있었던 부분도 확실히 
습득을 한 것 같다. 그리고 몰랐던 부분은 알아가는 계기가 된 것 같다.
만들면서 정말 어렵고 힘든 부분도 있었지만 알아가는 재미와 성취감으로 열심히 한 것 같다.
