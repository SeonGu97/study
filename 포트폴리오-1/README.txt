- 어제 말했던 것처럼 어사이드가 2개로 구성이 되어있다. 하지만 내용물은 같기 때문에 하나의 메뉴의 리스트를 클릭했을때 배경화면이 바뀐다고하면 다른 하나도 같이 적용이 되어야한다. 이 부분을 하려면, 하나의 엘리먼트의 이벤트 타겟을 클릭했을때, 다른 엘리먼트의 같은 위치에 있는 리스트도 가져와야한다. 그럴려면 공통적인 엘리먼트의 index를 가져와야하는데, 찾아보다가 index를 부여할 수 있어서, 반복문으로 양쪽의 어사이드 리스트의 인덱스를 부여하고 foreach 메서드의 index도 같이 사용하면서 해결했다.

- 어사이드 강조 효과 추가 (아이콘)

- 어제는 두개의 어사이드중에 하나의 어사이드 리스트가 클릭되어도 다른 하나의 같은 위치에 있는 리스트가 같이 변하는 기능을 추가했다면,
이번에는 같이 변하는 건 맞는데 같이 아이콘이 변해야한다. 그래서 생각대로 만들었는데 작동은 되는데 오류가 있었다.
아이콘의 클래스 이름이 "home" 이라면 이 아이콘은 속이 안채워진 홈이고 "home-fill" 은 속이 채워진 아이콘이다. 그래서 클릭을 한 리스트는 원래의 클래스 뒤에 "-fill"을 추가 하는 것이다.
그리고 다른 리스트를 누르면 "-fill"이라는 문자열을 모두 제거하고 다시 누른 리스트만 "-fill"을 추가하면 클릭했을때 배경도 강조되면서 아이콘의 색상이 채워지는 효과까지 되는것이다.
그런데 문제는 해당 클래스의 문자열을 가져와서 replace를 해도 안된다는 것이다. 어떻게 되냐면 클릭했을때 "home" 에서 "home=fill"이되고 한번더 누르면 "home-fill-fill"이 되는 것이다. 그래서 한번만 추가되고 다른걸 누르면 해당 "-fill"은 지워지게 하는게 목적이였는데,
계속 안되다가 생각한게 처음 replace를 한 문자열을 변수로 담아두고 replace 옵션에 새로운 문자열 부분에 넣는 것이다. 그래서 지금의 클래스를 old 부분에 넣고, replace로 저장해둔 클래스 문자열을 new 부분에 넣으면 완성이된다.
그렇게 해서 작동을 하니 잘 된다. 간단하게 금방 구현될 것 같았지만 어려움을 격은 기능이였다.
