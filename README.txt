- 오늘은 간단하게 class를 이용해서 이미지 슬라이더를 만들었다.
시간이 지나면 자동으로 넘어가는 슬라이더인데 포트폴리오에 채워 넣기 위해서
다양한 슬라이더를 많이 만들 예정이다.

(21.10.19)
- 오늘은 슬라이더를 애니메이션 효과로 만들어 보았다.
그 이유는 1부터5까지 슬라이더가 있는데, 1부터 시작해서 5까지 도달하면,
5에서 다시 1로 원상 복귀가 되야하는데 이 부분이 매끄럽게 연결이 되지 않아서,
고민을 하다가 애니메이션API가 있길래, 예제를 보고 따라 만들었더니 원하는 대로
만들어졌다.

(21.10.20)
- 오늘은 어제만든 슬라이더를 수정을 했다.
객체를 잘못 만들어서 다음 슬라이더를 넣을 수가 없었다. 그래서 코드는 그대로 쓰고
구조를 조금 바꾸었다.

(21.10.22)
- 오늘은 버튼을 누르면 왼쪽, 오른쪽으로 넘어가는 슬라이더를 만들었다.
역시 기존에 쓰던 방식과는 다르게 애니메이션으로 만들었다. 넘어가는 부분이 부드럽고
깔끔하다. appendChild, removeChild를 이용해서 firstChild와를 지우고 다시 appendChild를
이용해서 지운 첫번째 자식을 마지막으로 붙여줬다. 반대로는 flex-direction을 이용해서
row와 row-reverse를 이용해서 위치를 바꾸어주었다. 결론적으로 만족하는 슬라이더가 완성됬다.

(21.10.26)
- 오늘은 2번째로 만든 애니메이션의 수정할 부분이 있어서 해결방법을 모색해 보았다.
문제는 슬라이더가 작동은 정상적으로 되지만 슬라이더를 넘기는 버튼을 연타하면 넘어가는 모션을
생략한다는 것 이었다. 그래서 버튼을 누르더라도 슬라이더가 넘어가는 모션을 모두 마치고 다음으로
넘어가길 바랬는데 아직 애니메이션API는 익숙하지가 않아서 수정을 하지 못했다.

(21.10.27)
- 오늘은 슬라이더를 만들면서 부족하다고 느낀 애니메이션 부분을
조금더 기본부터 알아보려고 공부를 해봤다.
슬라이더를 만들면서 원하는 기능을 구현을 하지 못해서 찾아보다가
그냥 애니메이션을 공부를 해야겠다고 느꼈다. 대부분 만들다보면 이것
저것 갔다가 쓰면서 익숙해지고 사용방법을 알게되면서 자연스럽게
습득이 됬는데 애니메이션은 조금 깊게 공부를 해야 사용이 가능할 것 같아서
애니메이션을 공부하기로 했다.
